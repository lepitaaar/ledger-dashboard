# 장부 관리 디자인 시스템

## 1. 제품 원칙

장부 관리 UI는 장식보다 정확성, 상태 인지, 빠른 조작을 우선한다.

- 숫자와 상태를 가장 먼저 읽을 수 있게 한다.
- 같은 업무는 모든 화면에서 같은 위치와 표현을 사용한다.
- 색상만으로 상태를 구분하지 않고 라벨과 부호를 함께 제공한다.
- 데스크톱의 데이터 밀도와 모바일의 핵심 업무 수행 가능성을 모두 유지한다.
- 기존 API와 업무 규칙은 UI 개편과 분리한다.

## 2. 시각 토큰

토큰의 실제 값은 `src/app/globals.css`와 `tailwind.config.ts`를 기준으로 한다.

### 색상

- `background`: 전체 앱 배경. 밝은 회색 계열만 사용한다.
- `card`: 업무 콘텐츠 표면. 흰색과 얇은 경계선을 사용한다.
- `primary`: 주요 액션, 활성 메뉴, 링크에 사용하는 네이비 계열이다.
- `success`: 정상, 완료, 양수 상태에 사용한다.
- `warning`: 확인 필요, 미연결, 진행 대기 상태에 사용한다.
- `destructive`: 오류, 삭제, 재고 부족, 음수 금액에 사용한다.

화면별 임의 그라데이션이나 별도 브랜드 색상은 추가하지 않는다.

### 타이포그래피

- 기본 본문: `16px`
- 입력과 버튼: `14px`
- 데이터 표: `14px`
- 보조 문구와 상태: `12px`
- 페이지 제목: `24px`, 넓은 화면은 `28px`
- 금액과 수량: `tabular-nums`, 우측 정렬

### 크기와 간격

- 기본 입력 높이: `40px`
- 작은 버튼 높이: `32px`
- 기본 버튼 높이: `40px`
- 카드 모서리: `10px`
- 화면 섹션 간격: `24px`
- 카드 내부 여백: 기본 `20px`, 모바일 필터는 `16px`

## 3. 앱 셸

`AppShell`은 모든 `/dashboard` 화면에 적용한다.

- 데스크톱: 고정 사이드바, 그룹 메뉴, 접기 기능
- 모바일: 상단 메뉴 버튼과 오버레이 드로어
- 상단 바: 현재 페이지명, 오늘 날짜, 전역 빠른 작업
- 본문 최대 너비: `1560px`
- 검수 너비: `390px`, `768px`, `1440px`

메뉴 그룹 순서는 홈, 매출 관리, 상품 및 재고, 매입 연동, 분석으로 유지한다.

## 4. 표준 화면 구조

목록 및 분석 화면은 다음 순서로 구성한다.

1. `PageHeader`: 업무 영역, 제목, 설명, 주요 액션
2. `MetricCard` 그리드: 현재 화면의 핵심 숫자 3~4개
3. `FilterBar`: 기간, 검색, 선택 필터, 초기화
4. `ActiveFilterChips`: 현재 적용 조건
5. `DataTableShell`: 표, 모바일 목록, 빈 상태, 오류 상태
6. `Pagination`: 전체 건수, 현재 범위, 페이지 이동

페이지 제목을 카드 안에 다시 넣거나 주요 액션 위치를 화면마다 바꾸지 않는다.

## 5. 공통 컴포넌트

- `PageHeader`: 페이지당 한 번만 사용한다.
- `MetricCard`: 라벨, 값, 짧은 설명과 의미 있는 아이콘을 제공한다.
- `FilterBar`: 조회 조건을 한 영역에 모은다.
- `StatusBadge`: `neutral`, `success`, `warning`, `danger`, `info`만 사용한다.
- `DataTableShell`: 제목, 툴바, 표, 페이지네이션 영역을 통일한다.
- `MoneyText`: 금액 형식, 음수 색상, 숫자 폭을 통일한다.
- `EmptyState`: 데이터 없음과 검색 결과 없음을 구분해 설명한다.
- `ErrorState`: 오류 이유와 재시도 동작을 제공한다.

새 화면에서 유사 컴포넌트를 직접 만들기 전에 위 컴포넌트 확장을 우선한다.

## 6. 표와 모바일 목록

### 데스크톱

- 헤더는 한 줄을 기본으로 하고 스크롤 영역에서는 상단에 고정한다.
- 텍스트 열은 좌측, 금액과 수량은 우측, 상태와 작업은 중앙 또는 우측 정렬한다.
- 행 높이는 약 `44px`에서 `48px`를 유지한다.
- 삭제 등 파괴적 작업은 일반 작업과 색상 및 문구를 구분한다.

### 모바일

- 넓은 표를 축소하지 않고 카드형 행으로 전환한다.
- 제목, 핵심 금액, 상태를 첫 화면에 표시한다.
- 보조 숫자는 2~3열 요약 영역에 배치한다.
- 수정, 상세, 삭제 등 필수 작업을 숨기지 않는다.

## 7. 상태와 문구

- 성공: `정상`, `완료`, `매칭 완료`, `활성`
- 주의: `확인 필요`, `미매칭`, `원가 미연결`, `대기`
- 오류: `재고 부족`, `실패`, `손실`
- 비활성: `비활성`, `취소`, `삭제됨`

영문 장식 라벨은 사용하지 않는다. 날짜 기준은 화면에 직접 하드코딩하지 않고 API 메타데이터나 공통 설정에서 가져온다.

## 8. 접근성과 검수

- 모든 아이콘 버튼에 `aria-label`을 제공한다.
- 키보드 포커스는 `focus-visible` 링으로 확인할 수 있어야 한다.
- 상태는 색상뿐 아니라 텍스트로도 전달한다.
- 긴 업체명, 큰 금액, 데이터 없음, 로딩, API 오류를 확인한다.
- `npm run typecheck`, `npm run lint`, `npm run test`, `npm run build`를 실행한다.
- 주요 화면은 `390px`, `768px`, `1440px`에서 시각 검수한다.
